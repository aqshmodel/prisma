## 1. ファイルの基本仕様

### ファイル形式・保存先
| 項目 | 仕様 |
|---|---|
| ファイル形式 | `.mdx`（MDX: Markdown + JSX） |
| 保存先 | `content/articles/` |
| ファイル名 | 英単語ケバブケース（例: `infp-want-to-quit-job.mdx`） |
| ❌ 禁止 | ローマ字混じり（例: `infp-shigoto-yametai.mdx`） |

### Frontmatter（必須フィールド）

すべての記事の冒頭に以下のYAML Frontmatterを必ず記述する。

```yaml
---
title: 'INFJが静かに壊れる理由──職場の人間関係に疲れたときの3つの処方箋'
description: '記事のメタディスクリプション。120〜160文字。検索結果に表示される。'
date: '2026-03-01'
coverImage: '/images/blog/infj-work-relationships-exhaustion.jpg'
author: 'Aqsh Prisma 編集部'
category: '働き方・キャリア'
tags: ['INFJ 仕事 人間関係 疲れる', 'INFJ 職場 共感疲労', 'ソシオニクス']
slug: 'infj-work-relationships-exhaustion'
---
```

| フィールド | 説明 | 注意点 |
|---|---|---|
| `title` | 記事タイトル（h1） | 下記「タイトルの文字数ルール」を厳守 |
| `description` | メタディスクリプション | 120〜160文字。ターゲットKWを自然に含める |
| `date` | 公開日 | `YYYY-MM-DD` 形式 |
| `coverImage` | カバー画像パス | `/images/blog/{slug}.jpg` |
| `author` | 著者名 | `Aqsh Prisma 編集部` を統一使用 |
| `category` | カテゴリ | 下記「カテゴリ一覧」参照 |
| `tags` | タグ（ロングテールKW） | メインKW + サブKW + 関連キーワード。配列形式 |
| `slug` | URL用スラッグ | ファイル名と同一にすること（既存の例外は2件のみ） |

### カテゴリ一覧（統一ルール）

| カテゴリ名 | 使い分け |
|---|---|
| `働き方・キャリア` | 仕事・転職・職場の人間関係・会議・マネジメント等 |
| `恋愛・人間関係` | 恋愛・夫婦・友人・親子・苦手な人等 |
| `メンタルヘルス` | ストレス・バーンアウト・自己肯定感・HSP等 |
| `自己分析・診断` | 16タイプ入門・エニアグラム入門・MBTI比較等 |

## 2. タイトル・見出しの文字数ルール（ハイブリッド構造）

| 要素 | 文字数 | 役割 | 例 |
|---|---|---|---|
| **タイトル前半（フック）** | **13文字以内** | 一瞬で心をつかむキャッチコピー | `INFJが静かに壊れる理由` |
| **タイトル全体（h1）** | **30〜35文字** | Google検索結果で切れないSEO最適化 | `INFJが静かに壊れる理由──職場の人間関係に疲れたときの3つの処方箋` |
| **記事内見出し（h2/h3）** | **13文字以内** | スクロール中に目に飛び込む短い見出し | `共感疲労が生まれる構造` |

**タイトル構造**: `【13文字フック】──【SEO補足】` の2パート構造を基本とする。

> ⚠️ 見出しの文字数カウントは全角・半角関係なく「文字数」で数えること。

## 3. 記事本文の構成テンプレート

### 基本構成（悩み解決型 [Do] インテント）

```
## h2: [共感・問題提起]（13文字以内）
  本文: 読者の「あるある」を描写。セリフや場面描写で引き込む

## h2: [原因の構造解説]（13文字以内）
  ### h3: 心理機能Aの解説（13文字以内）
  ### h3: 心理機能Bの解説（13文字以内）
  ### h3: エニアグラムとの重なり（13文字以内）

## h2: [解決策・処方箋]（13文字以内）
  ### h3: 対策1（13文字以内）
  ### h3: 対策2（13文字以内）
  ### h3: 対策3（13文字以内）

※本記事は自己分析のフレームワークであり、医療的アドバイスではありません。

<DiagnosisCTA />
```

### 導入パターンのバリエーション（記事間の差別化）

同じ構成パターンでも導入の書き出しを変えて、ブログ全体に変化をつける。

| パターン | 例（冒頭の1文目） | 適用例 |
|---|---|---|
| **セリフ型** | 「別にいいよ、私がやっておくから」── 今日もまた… | INFJ仕事疲れ |
| **場面描写型** | 金曜の夜の飲み会。視界の端で一人だけ… | ENFJ空気読みすぎ |
| **データ/事実型** | ある調査では大学生の約70%が「自分は先延ばしグセがある」と… | 先延ばし癖 |
| **読者への問いかけ型** | マッチングアプリで何人と会っても、すぐに「この人じゃない」と… | INFP恋愛理想 |

## 4. 本文のライティングルール

### 文体・トーン
- **話し言葉と書き言葉のミックス**: 「〜である」「〜だ」の断定ベースに「〜だよね」「〜じゃないか」を適度に混ぜる
- **ターゲット読者**: 20代女性でも読みやすいこと。難読漢字はひらがなに開く
  - ❌ `宥めよう` → ⭕️ `なだめよう`
  - ❌ `諫める` → ⭕️ `いさめる`
- **1段落は3〜4行以内**: スマホでの可読性を最優先

### 装飾ルール
| 装飾 | ルール |
|---|---|
| **太字（`**`）** | 1段落に最大1箇所。キラーフレーズ（共感ポイント・結論部分）に限定 |
| 箇条書き | 最小限に。文脈で読ませる構成を優先 |
| 引用（`>`） | 使わない（読者の声は本文に溶かす） |

### AI臭さの排除チェックリスト
以下のフレーズ・パターンが含まれていたら**必ず削除・書き換え**する。

- ❌ 「結論から言うと」「まず〜について解説します」
- ❌ 「いかがでしたか？」「最後に一言」
- ❌ 「以下の通りです」「次に解説します」
- ❌ 冒頭の過剰な共感（「〜で悩んでいませんか？大丈夫です！」）
- ❌ 各セクション冒頭の「ここでは〜を説明します」
- ❌ 装飾の乱用（太字3箇所以上、絵文字の多用）

## 5. 内部リンクのルール

### 必須の内部リンク
| 設置箇所 | リンク先 | 本数目安 |
|---|---|---|
| 本文中（自然な文脈） | 同クラスター内の関連記事 | 2〜3本 |
| 記事末尾 | `<DiagnosisCTA />` コンポーネント | 必ず1つ |
| 本文中 | ピラーページ（入門記事） | 1本 |

### アンカーテキストのルール
- ❌ 「詳細はこちら」「この記事を読む」
- ⭕️ リンク先のターゲットKWまたは記事の内容を含む短文
- 例: `[ISFJが職場で断れない理由](/articles/isfj-cannot-say-no-at-work)`

### slug参照
内部リンクのURLは `/articles/{slug}` 形式。slug一覧は [記事テーマサマリ](./記事テーマサマリ.md) のslug対応表を参照。

## 6. カバー画像の仕様

| 項目 | 仕様 |
|---|---|
| 保存先 | `public/images/blog/` |
| ファイル名 | `{slug}.jpg` |
| 画像幅 | 1200px |
| ファイルサイズ | **100KB以下**（`sips` コマンドでリサイズ可能） |
| アスペクト比 | 16:9 推奨 |
| 内容 | 実写の20代日本人女性。記事テーマの心理状態を反映した表情・シーン |

### リサイズコマンド（macOS）
```bash
sips -s format jpeg -s formatOptions 40 -Z 1200 元画像.jpg --out public/images/blog/{slug}.jpg
```

## 7. CTA（診断誘導）ルール

### コンポーネント仕様
記事内の診断CTAは、**テキストリンクではなく共通のボタンコンポーネント `<DiagnosisCTA />`** を使用する。

| 項目 | 仕様 |
|---|---|
| コンポーネント | `<DiagnosisCTA />` |
| 実装ファイル | `src/features/articles/components/DiagnosisCTA.tsx` |
| MDX登録場所 | `src/app/articles/[slug]/page.tsx` の `components` オブジェクト |
| リンク先 | `/diagnosis` |
| ボタンテキスト | 「FREE ANALYSIS / 無料で診断する →」 |

### 記事内での配置ルール
1. **記事末尾に必ず1つ配置する**（免責事項の直後）
2. **本文中にインラインのテキストリンク（`[性格診断テスト](/diagnosis/)`）は設置しない**。診断への誘導はすべてボタンに任せる
3. 本文のクロージング段落では、診断を直接勧めるのではなく「自分のOSを特定しよう」「脳の仕様を知ろう」といった**感情的な動機づけ**に留め、ボタンへの橋渡しとする
4. `---`（水平線）で本文と区切り、その直後に `<DiagnosisCTA />` を配置する

### 標準的な記事末尾テンプレート
```mdx
（感情的なクロージング段落。ここでは「診断を受ける」とは言わない）

※本記事は自己分析のフレームワークであり、医療的アドバイスではありません。

<DiagnosisCTA />
```

### 診断の使い分け（本文の文脈で）
- **キャリア・コミュニケーション系** → 「脳のOS」「情報処理パターン」を特定しよう、という文脈で誘導
- **メンタルヘルス・モチベーション系** → 「心のエンジン」「駆動する欲求」を知ろう、という文脈で誘導

## 9. 公開前チェックリスト

記事を公開する前に、以下をすべて確認すること。

- [ ] h1タイトルのフックは**13文字以内**か？全体は**30〜35文字**か？
- [ ] h2/h3見出しはすべて**13文字以内**か？
- [ ] Frontmatterの全フィールド（title, description, date, coverImage, author, category, tags, slug）が埋まっているか？
- [ ] `description` は**120〜160文字**でターゲットKWを含んでいるか？
- [ ] カバー画像が `public/images/blog/{slug}.jpg` に配置済みで**100KB以下**か？
- [ ] AI臭いフレーズが残っていないか？（セクション4のチェックリスト参照）
- [ ] 太字装飾が1段落1箇所以内に収まっているか？
- [ ] 内部リンクが**2〜3本**以上、自然な文脈で設置されているか？
- [ ] 記事末尾に `<DiagnosisCTA />` コンポーネントが設置されているか？
- [ ] 記事末尾に**免責事項**が記載されているか？
- [ ] ファイル名（slug）が [記事テーマサマリ](./記事テーマサマリ.md) に登録されているか？
